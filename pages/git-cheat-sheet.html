GIT CHEAT SHEET

CREATE

Clone an existing repository
Клонировать существующий репозиторий

$ git clone ssh://user@domain.com/repo.git

Create a new local repository
Создать новый локальный репозиторий

$ git init
LOCAL CHANGES

Changed files in your working directory
Изменённые файлы в вашем рабочем каталоге

$ git status

Changes to tracked files
Изменения в отслеживаемых файлах

$ git diff

Add all current changes to the next commit
Добавить все текущие изменения в следующий коммит

$ git add .

Add some changes in 'file' to the next commit
Добавьте некоторые изменения в 'file' к следующему коммиту

$ git add -p 'file'

Commit all local changes in tracked files
Зафиксируйте все локальные изменения в отслеживаемых файлах

$ git commit -a

Commit previously staged changes
Зафиксируйте ранее поставленные изменения

$ git commit

Change the last commit
Don‘t amend published commits!
Изменить последний коммит
Не изменяйте опубликованные коммиты!

$ git commit --amend
COMMIT HISTORY

Show all commits, starting with newest
Показать все коммиты, начиная с самых новых

$ git log

Show changes over time for a specific file
Показать изменения с течением времени для конкретного файла

$ git log -p 'file'

Who changed what and when in 'file'
Кто что изменил и когда в 'file'

$ git blame 'file'
HELP & DOCUMENTATION

Get help on the command line
Получить справку в командной строке

$ git help 'command'
BRANCHES & TAGS

List all existing branches
Перечислите все существующие филиалы

$ git branch -av

Switch HEAD branch
Переключатель ГОЛОВНОЙ филиал

$ git checkout 'branch'

Create a new branch based on your current HEAD
Создать новую ветку на основе на вашей текущей голове

$ git branch 'new-branch'

Create a new tracking branch based on a remote branch
Создать новую ветку отслеживания на основе удаленная ветка

$ git checkout --track 'remote/branch'

Delete a local branch
Удалить локальную ветку

$ git branch -d 'branch'

Mark the current commit with a tag
Пометить текущий коммит тегом

$ git tag 'tag-name'
UPDATE & PUBLISH

List all currently configured remotes
Список всех текущих настроенных пультов

$ git remote -v

Show information about a remote
Показать информацию о пульте

$ git remote show 'remote'

Add new remote repository, named 'remote'
Добавить новый удаленный репозиторий с именем 'remote'

$ git remote add 'shortname' 'url'

Download all changes from 'remote', but don‘t integrate into HEAD
Загрузите все изменения из 'remote', но не интегрируйте в HEAD

$ git fetch 'remote'

Download changes and directly merge/integrate into HEAD
Скачать изменения и напрямую объединить / интегрировать в ГОЛОВУ

$ git pull 'remote' 'branch'

Publish local changes on a remote
Публиковать локальные изменения на удаленном

$ git push 'remote' 'branch'

Delete a branch on the remote
Удалить ветку на пульте

$ git branch -dr 'remote/branch'

Publish your tags
Опубликуйте свои теги

$ git push --tags
MERGE & REBASE

Merge 'branch' into your current HEAD
Слить «ветку» в вашу текущую ГОЛОВУ

git merge 'branch'

Rebase your current HEAD onto 'branch'
Don‘t rebase published commits!
Перенесите вашу текущую ГОЛОВКУ на <филиал>
Не перебазируйте опубликованные коммиты!

$ git rebase 'branch'

Abort a rebase
Прервать ребаз

$ git rebase --abort

Continue a rebase after resolving conflicts
Продолжить ребаз после разрешения конфликтов

$ git rebase --continue

Use your configured merge tool to solve conflicts
Используйте настроенный инструмент слияния для разрешать конфликты

$ git mergetool

Use your editor to manually solve conflicts
and (after resolving) mark file as resolved
Используйте свой редактор для ручного разрешения конфликтов и (после разрешения) пометить файл как разрешенный

$ git add 'resolved-file'
$ git rm 'resolved-file'
UNDO

Discard all local changes in your working directory
Откажитесь от всех локальных изменений в вашей работе каталог

$ git reset --hard HEAD

Discard local changes in a specific file
Отменить локальные изменения в определенном файле

$ git checkout HEAD 'file'

Revert a commit (by producing a new commit with contrary changes)
Отменить коммит (создав новый коммит с противоположными изменениями)

$ git revert 'commit'

Reset your HEAD pointer to a previous commit
…and discard all changes since then
Сброс указателя HEAD на предыдущий коммит
... и отменить все изменения с тех пор

$ git reset --hard 'commit'

…and preserve all changes as unstaged changes
… И сохранить все изменения как незакрепленные изменения

$ git reset 'commit'

…and preserve uncommitted local changes
… И сохранить незафиксированные локальные изменения

$ git reset --keep 'commit'
